Tableau des balises html et attributs

Présentation :

Le HyperText Markup Language, généralement abrégé html ou dans sa dernière version html5, est le langage de balisage conçu pour représenter les pages web. Créé en 1991 c'est un langage qui permet de définir les différents contenus et de créer la structure d’une page. L’anglais Hypertext Markup Language se traduit littéralement en langage de balisage d’hypertexte1. On utilise généralement le sigle html, parfois même en répétant le mot « langage » comme dans « langage html ». Hypertext est parfois écrit HyperText pour marquer le T du sigle html.
Origine :
html est une des trois inventions à la base du World Wide Web, avec le Hypertext Transfer Protocol (HTTP) et les adresses web (URL). html a été inventé pour permettre d'écrire des documents hypertextuels liant les différentes ressources d’Internet avec des hyperliens. Aujourd’hui, ces documents sont appelés « page web ».
Le public non averti parle parfois de HTM au lieu de html, HTM étant l’extension de nom de fichier tronquée à trois lettres, une limitation que l’on trouve sur d’anciens systèmes d’exploitation de Microsoft.

Syntaxe html :

elements balises fonctions
<!DOCTYPE> Sert à indiquer le type de document.
<html> Balise d'entête dans laquelle l'ensemble du code va être contenu.
<!--.........--> Permet d'ajouter un commentaire dans un code de page html.
<head> Elément d’en-tête va contenir des éléments qui vont servir à fournir des informations sur la page au navigateur, comme le titre de la page ou encore le type d’encodage.
<meta> Sert lui à transmettre des metas informations sur la page au navigateur. Le type d’information transmis va dépendre de l’attribut que l’on va préciser et le type d’encodage utilisé dans nos pages. Cela va permettre aux navigateurs d’afficher correctement nos différents textes. Pour faire cela, nous allons utiliser l’attribut charset pour « characters set » c’est-à-dire « ensemble de caractères » de l’élément meta et lui fournir la valeur utf-8
<link="...."> Placer dans l'élément head, les links seront lu et appliqués dans notre page ou fichier. Les attributs rel et href vont être utilisés dans cette balise.css .
<title> L’élément title va nous permettre d’indiquer le titre de la page.Il ne doit pas être confondu avec les différents textes définis comme des titres dans la page. Ce titre de page est le texte visible sur le haut des onglets de votre navigateur.
<body> Le body va contenir l'ensemble des éléments visible de la page.
<h1> C'est le titre principal de la page. Il ne peut y en avoir qu'un par page.
<p> On peut créer autant de paragraphes que l’on souhaite dans une page. A chaque nouveau paragraphe, il faut utiliser un nouvel élément p. On peu lui attribuer différentes class afin de le cibler précisement.
<br> Est utilisé pour les retour à la ligne. C'est une Balise orpheline.
<hr> Est utilisé pour un retour à la ligne avec en plus une ligne séparatice. C'est une Balise orpheline.
<pre> L’élément pre sert à préformater un texte. Cela signifie que tout le contenu qui se trouve à l’intérieur de cet élément va conserver la mise en forme que nous allons lui donner lors du rendu fait par le navigateur.
<strong> Element qui sert à definir une partie très importante dans un texte et sera considéré par les moteurs de recherche. Resultat visuel en caractère gras.
<em> Element qui sert à définir une partie importante dans un texte. Considéré par les moteurs de recherches. Résultat visuel en italic.
<mark> Element qui sert a surligner la partie qui est a l'interieur.
<ul> Element de liste non ordonnée par des puces qui va contenir d'autres éléments de balisage..
<ol> Element de liste ordonnée par des chiffres.Balisage dans lequel on va pouvoir imbriquer d'autres éléments de balisage. Les attributs start ou value peuvent lui être ajouté ainsi que reversed. Un style de type= A, I peu lui être attribué afin définir le type de numérotation.
<li> Element de liste ordonnée.
<a> Va être utilisé afin d'insérer un lien hypertext avec l'attribut "href". L'attribut target qui permet de cibler où doit s’ouvrir notre page de destination. La valeur _blank> va spécifier que la nouvelle page doit s’ouvrir dans un nouvel onglet. On peu également lui attributer des éléments d' id ou de class afin de cible précisement cette balise et la mettre en forme en css .
L'attribut mailto servira afin d'inserer un lien pour pouvoir envoyer un email.
<div> Balisage et élément de type block.
<span> Va être utilisé afin de selectionner une partie précise du texte pour pouvoir lui appliquer un style css particulier. Elément de type inline.
<table> Balisage qui va définir un tableau. Contiendra les attributs "th", "tr" et "td" qui permettent de définir les colonnes et les lignes.
<caption> Afin d'ajouter une legende (ou titre) à un tableau, cette balise doit être placé après un <table>. Caption-side en css pour déterminer ou va se situer la legende.
<tr> Va permettre d’ajouter des lignes dans un tableau.
<td> Va permettre d'ajouter des colonnes à un tableau.
<th> Pour créer une ligne d’en-tête d'un tableau. Dans un tableau complexe, 'scope="col" ' défini un sens de lecture vertical et 'scope="row" ' un sens de lecture horizontal.
<thead> Balisage qui va définir l'entête d'un tableau.
<tbody> Balisage qui va définir le corps d'un tableau.
<tfoot> Balisage qui va définir le pied d'un tableau.
<col> et <colgroup> Dans un tableau complexe, balisage qui permettera de definir le nombre de colonnes ou lignes à regrouper.
<img.> Va permettre d'insérer une image dans une page. Suivis de l'attribut 'src="..." ' pour indiquer le chemin à cette balise et de Alt="décrire l'image" servira à décrire l'image. Les éléments figure et figcaption en css vont permettre d'afficher le commentaire en dessous pour les non voyants ou autres.
<svg> Va permettre d'insérer une image vectorielle ou dessin. Lien vers les contenus autorisés.
<video controls> Inclut les éléments universels comme 'src="..." ' ou encore 'autoplay="false" '. Lien vers la documentation d'intégration de contenu vidéo.
<audio controls> Va permettre d'insérer de l'audio dans la page.
<audio controls
'src="....." '' pour indiquer le chemin à cette balise et 'type="....." ' qui va permettre d'indiquer au navigateur le codec audio utilisé dans notre fichier ( ex: audio/mpeg ).
</audio>
<form> Balise qui contient la sémantique nécessaire à la construction d'un formulaire. L' attributs ' action=" ' traîte l'envoie du formulaire" ' et ' method"méthode utilisée pour envoyer les données au serveur" '.Lien vers la documentation de structure d'un formulaire.
<fieldset> Regroupe l'ensemble des champs nécessaires à la création de formulaires. L'attribut ' role="group" ' permet la même chose
<label> Balise orpheline utilisé pour définir une zone de texte suivis d'un for="information identique à celle de l'input" ' 'aria-labelledby=" assistance qui décrit la nature de l'objet" '. Le label doit toujours être liée à un champ "input"
<input> Est utilisé pour créer un contrôle interactif dans un formulaire, dépend de la valeur indiquée dans son attribut 'type="button"' ou 'type="submit"', 'onclick="alert('alert')"' permet de tester si le bouton fonctionne et 'value="envoyer"'. Lien vers la documentation de structure d'un bouton.
<textarea> Permet d'éditer un texte sur plusieurs lignes dans un formulaire.
<footer> Défini une boite pour un bas de page.
